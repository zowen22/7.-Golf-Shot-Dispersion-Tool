## 2026-06-17 - Golfbert API Fixes + Round Lifecycle

*Status: `COMPLETED`*
*Type: `Manual`*

### What Was Done
- Rewrote `GolfbertModels.swift` to match actual Swagger spec: `resources` envelope (not `data`), Int IDs, `long` not `lng`, explicit CodingKeys for concatenated-lowercase fields (`courseid`, `flagcoords`, `holeid`, `surfacetype`), teeboxes and polygons as separate endpoints per hole
- Confirmed Golfbert uses AWS Signature V4 (from their sample request); implemented `GolfbertSigner.swift` using CryptoKit — zero new SPM dependencies
- Updated `NetworkService`: `?name=` search param, `&long=` geo param, parallel course detail + teebox fetch, 3-call hole fetch flow
- Added `fetchAllHoles(courseId:)` batch method to `NetworkService`: 1 call for hole list + N×2 parallel teebox/polygon calls, replaces N×3 individual calls that were firing from `HoleSelectorView`
- Created `ActiveRoundViewModel`: creates Round doc in Firestore on session start, tracks `holesPlayed` as user navigates holes
- Wired round lifecycle: HoleSelectorView creates `ActiveRoundViewModel` as `@StateObject`, calls `roundVM.start()` on `.task`, passes `roundVM` to HoleMapView
- Fixed pre-existing HoleMapView bug: `holeNumber` was a `let` constant so prev/next navigation never updated the hole title or button disabled states; replaced with `@State private var currentHoleNumber`
- Updated Work Packages (progress counter, new tasks for sessions 1–2)

### Decisions Made
- API key doubles as AWS Credential per Golfbert's own example; separate secret key required for HMAC — both stored in gitignored `Secrets.xcconfig`
- Round is created in `HoleSelectorView` (when hole selection begins), not `TeeSelectionView`, to keep TeeSelectionView as pure UI; Round is created for both `.round` and `.study` modes
- Batch prefetch falls back to individual `fetchHole` calls only for Firestore cache hits; a single `fetchAllHoles` handles all cache misses regardless of count

### Blockers / Issues
- All Firebase, Mapbox, and Golfbert wiring still @user tasks (requires Xcode)
- `GOLFBERT_SECRET_KEY` must be added to Info.plist in Xcode (documented in XCODE_SETUP.md)

- First build: uncomment Firebase imports, validate auth flow
- Validate Golfbert API responses against models with real API key
